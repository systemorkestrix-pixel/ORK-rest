# المرجع الداخلي للنموذج التشغيلي

هذه الوثيقة مرجع داخلي للفريق لفهم المنطق التشغيلي الحقيقي للمطعم، وتوزيع الملكية بين اللوحات والأدوار والحالات دون خلط بين الإدارة والتنفيذ والرقابة.

## 1) الهدف
- تثبيت منطق تشغيلي واحد يبنى عليه التصميم والواجهات والصلاحيات.
- منع خلط الأعمال الإدارية مع الأعمال التشغيلية اليومية.
- توضيح مالك كل إجراء، ومالك كل حالة طلب، ومالك كل انتقال داخل النظام.

## 2) الطبقات التشغيلية المعتمدة

### أ. طبقة الإدارة
هذه الطبقة تضبط النظام ولا تنفذ الطلبات بنفسها.

أمثلة الأدوار:
- مدير النظام
- مدير الفرع
- مسؤول النظام
- المحاسب الإداري
- مسؤول المخزون

الملكية:
- المستخدمون
- الأدوار والصلاحيات
- الإعدادات العامة
- الأسعار والسياسات
- رسوم التوصيل
- الموردون
- التقارير
- سجل التدقيق

لا تملك:
- بدء التحضير
- تسليم الطلب
- استلام مهمة التوصيل
- تنفيذ العمل الميداني

### ب. طبقة التشغيل اليومي
هذه الطبقة تدير تدفق العمل اليومي داخل المطعم، وهي الطبقة التي غالبًا يحدث فيها الخلط إذا لم تُفصل عن الإدارة.

أمثلة الأدوار:
- الكاشير
- مسؤول التشغيل
- مسؤول الشفت
- موظف استقبال الطلبات

الملكية:
- إنشاء الطلب
- تعديل الطلب قبل اعتماده
- ربط الطلب بطاولة أو عميل
- متابعة الطلبات المفتوحة
- تحويل الطلب إلى المسار المناسب
- إغلاق الطلب بعد اكتمال الدورة
- بدء التسوية أو متابعة التعثر

لا تملك:
- إدارة المستخدمين
- تعديل الإعدادات العامة
- إدارة الصلاحيات
- تنفيذ الطبخ أو التوصيل بنفسها إلا عند غياب الأنظمة المنفصلة وبوضع صريح

### ج. طبقة التنفيذ
هذه الطبقة تنفذ فقط، ولا تملك أدوات الإدارة.

أمثلة الأدوار:
- موظف المطبخ
- قائد المطبخ
- سائق التوصيل

الملكية:
- استلام مهمة التنفيذ
- تغيير حالة التنفيذ
- إتمام المهمة
- الإبلاغ عن التعذر أو الفشل

لا تملك:
- إدارة النظام
- المستخدمين
- الأسعار العامة
- المخزون الإداري الكامل
- التقارير الإدارية

### د. طبقة الدعم والرقابة
هذه الطبقة تدعم التنفيذ وتراجع نتائجه، لكنها ليست بديلاً عن التشغيل اليومي.

أمثلة الأدوار:
- المالية
- المخزون
- التدقيق
- التحليلات

الملكية:
- التسويات
- الحركات المالية
- حركة المخزون
- المطابقة والمراجعة
- اكتشاف الانحرافات
- تقارير الأداء

## 3) اللوحات المعتمدة

### Manager Console
لوحة الإدارة والتشغيل اليومي.

تحتوي:
- العمليات
- المطعم
- إدارة التوصيل
- المخزون
- المالية
- التحليلات
- النظام

ملاحظة:
هذه اللوحة قد تحتوي أدوات تشغيل يومي مثل إنشاء الطلب ومتابعته، لكنها لا يجب أن تتحول إلى لوحة تنفيذ للمطبخ أو السائق.

### Kitchen Console
لوحة تنفيذ مستقلة للمطبخ.

تحتوي فقط:
- قائمة الطلبات المطلوب تحضيرها
- حالات التحضير
- الجاهزية
- التعذر

### Delivery Console
لوحة تنفيذ مستقلة للسائق.

تحتوي فقط:
- الطلبات المتاحة
- المهام المسندة
- الانطلاق
- التسليم
- الفشل

### Public Interface
واجهة العميل أو الواجهة العامة.

تحتوي فقط:
- استعراض المنتجات
- إنشاء الطلب العام
- عرض حالة الطاولات أو الخدمات العامة عند الحاجة

## 4) الدور الناقص الواجب اعتماده
الدور التشغيلي الناقص منطقيًا هو:
- مسؤول التشغيل

هذا الدور هو الفاصل بين الإدارة والتنفيذ.

ملكيته:
- إدارة الطلبات اليومية
- متابعة حالات الطلب
- التدخل عند التعثر
- التنسيق بين الكاشير والمطبخ والتوصيل
- فتح وإغلاق الدورة التشغيلية اليومية للطلب

هذا الدور لا يملك:
- إدارة المستخدمين
- الإعدادات العامة
- الصلاحيات المؤسسية
- سجل التدقيق

## 5) دورة الطلب القياسية

### طلب صالة
- إنشاء الطلب
- ربط الطلب بالطاولة
- اعتماده للتنفيذ
- التحضير أو التجهيز المباشر
- التسليم للطاولة
- إغلاق الجلسة
- التسوية

### طلب سفري
- إنشاء الطلب
- اعتماده للتنفيذ
- التحضير
- الجاهزية للاستلام
- التسليم للعميل
- التسوية

### طلب توصيل
- إنشاء الطلب
- اعتماده للتنفيذ
- التحضير
- الجاهزية للتوصيل
- إسناد المهمة للسائق
- الانطلاق
- التسليم أو الفشل
- التسوية

## 6) ملكية الحالات والانتقالات

### حالات ما قبل التنفيذ
مالكها:
- التشغيل اليومي

تشمل:
- جديد
- قيد المراجعة
- مؤكد
- ملغى قبل التنفيذ

### حالات المطبخ
مالكها:
- المطبخ

تشمل:
- قيد التحضير
- جاهز
- متعذر التحضير

### حالات التوصيل
مالكها:
- السائق أو نظام التوصيل التنفيذي

تشمل:
- مسند
- في الطريق
- تم التسليم
- فشل التوصيل

### حالات الإغلاق والتسوية
مالكها:
- التشغيل اليومي مع دعم المالية

تشمل:
- بانتظار التسوية
- مسوى
- مغلق

## 7) القاعدة الذهبية للملكية
كل خطوة في النظام يجب أن يكون لها ثلاثة ملاك واضحين:
- مالك قرار
- مالك تنفيذ
- مالك رقابة

مثال:
- قبول الطلب: التشغيل اليومي
- تحضيره: المطبخ
- توصيله: السائق
- تسويته: التشغيل اليومي أو المالية
- مراجعته: التدقيق

## 8) مواضع العناصر داخل النظام

### ما يبقى داخل لوحة الإدارة
- إنشاء الطلب
- متابعة الطلبات المفتوحة
- الطاولات
- إدارة التوصيل
- إدارة الرسوم والسياسات
- المخزون
- المالية
- التحليلات
- النظام

### ما يخرج إلى لوحة المطبخ
- بدء التحضير
- تحديث مراحل التحضير
- إعلان الجاهزية
- إعلان التعذر

### ما يخرج إلى لوحة السائق
- استلام المهمة
- الانطلاق
- التسليم
- فشل التوصيل

## 9) القواعد التي تمنع التكرار
- أي عنصر يضبط النظام بالكامل يبقى في الإدارة.
- أي عنصر ينفذ مهمة حية يذهب إلى لوحة التنفيذ.
- أي عنصر يراجع أو يراقب يبقى في الرقابة أو التحليلات.
- لا تكرر الإجراء نفسه في أكثر من لوحة إلا إذا كانت إحدى النسختين للعرض فقط.

## 10) أثر هذا المرجع على الواجهة
- الهيدر يعرض المتابعة السريعة فقط، وليس بديلًا عن صفحات الإدارة.
- التنبيهات تبقى مركزًا للإجراءات والمتابعة.
- حالة الاستقرار تصبح سجلًا تشغيليًا منفصلًا ومختصرًا.
- حساب المدير يصبح ملكية شخصية مستقلة عن إدارة المستخدمين.

## 11) القرار المعتمد للفريق
- اعتماد فصل صريح بين الإدارة، التشغيل اليومي، التنفيذ، والرقابة.
- اعتماد مسؤول التشغيل كدور منطقي أساسي.
- عدم تحميل المدير كل أعمال التشغيل والتنفيذ.
- عدم خلط إدارة التوصيل مع تنفيذ التوصيل.
- عدم خلط إدارة المطبخ مع تنفيذ المطبخ.
