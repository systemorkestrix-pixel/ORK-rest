# مهام تنفيذية: إعادة بناء الواجهات بعد التصحيح المعماري

هذه المهام تعكس الخطة المصححة (فصل التطبيقات التشغيلية عن لوحة الإدارة) مع الحفاظ على التصميم الحالي للـ Manager Console.

## 0) تثبيت الأساس المعماري
- [x] اعتماد مسارات التطبيقات الأربعة: `/console/*`, `/kitchen/*`, `/delivery/*`, `/public/*`.
- [x] تحديث Login و RoleGuard لكل تطبيق بشكل مستقل.
- [x] منع خلط مسارات Delivery Console مع أقسام Manager Console.
- [ ] مراجعة أي Redirects قديمة متبقية والتأكد أنها لا تعيد توجيه لوحات التشغيل إلى لوحة الإدارة.

## 1) تنظيم Manager Console (الأقسام الإدارية)
- [x] تحديث ChannelBar لتعكس الأقسام الإدارية الجديدة.
- [x] تحديث ConsolePage للربط مع `/console/*` فقط.

- [ ] تدقيق جميع روابط الصفحات الإدارية لتستخدم `/console/*` بدل المسارات القديمة.

## 2) فصل لوحات التشغيل (Kitchen/Delivery)
- [x] تثبيت Kitchen Console تحت `/kitchen/console` وربط الصفحات التشغيلية.

- [ ] تدقيق أي روابط داخلية في صفحات التشغيل لضمان عدم انتقالها إلى Manager Console.

## 3) توحيد الصفحات والملكية
- [ ] مراجعة كل صفحة لتأكيد مالكها (Manager أو Kitchen أو Delivery).

- [ ] ربط الصفحات بالـ entities المناسبة إن كانت لا تزال تستخدم API مباشرة.
