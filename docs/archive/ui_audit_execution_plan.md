# خطة تنفيذ إصلاحات التقارير (UI Audit + UI Logic Audit)

المرجع:
- `docs/ui_audit_report.md`
- `docs/ui_logic_audit_report.md`

## قواعد التنفيذ
- لا نبدأ إصلاحًا جديدًا إلا بعد إغلاق السابق بالكامل (تنفيذ + تحقق + تنظيف موضعي).
- التسوية تبقى في العمليات وفق قرار المستخدم (سهولة التشغيل).
- الإصلاح يتم على مرحلتين (Turn 1, Turn 2) ثم تنظيف شامل.

## المرحلة 1 (Turn 1)
1. إعدادات التوصيل: نقل واجهة إعدادات التوصيل (رسوم/حد أدنى/تبليغ تلقائي) من النظام إلى قناة التوصيل الإدارية.
   - إنشاء قسم جديد داخل `/console/delivery/*`.
   - تعديل الروابط التي تشير لإعدادات التوصيل داخل التوصيل.
   - التحقق: الوصول من قناة التوصيل إلى إعدادات التوصيل دون المرور بالنظام.

2. عناوين الصفحات المفقودة بعد إزالة Header العام:
   - إضافة عناوين رئيسية واضحة في:
     - `src/modules/intelligence/reports/ReportsPage.tsx`
     - `src/modules/intelligence/audit/AuditLogsPage.tsx`
   - التحقق: ظهور عنوان واضح لكل صفحة مع الحفاظ على التصميم العام.

## المرحلة 2 (Turn 2)
3. فصل بيانات تشغيل السائق عن إدارة المستخدمين العامة:
   - إبقاء إنشاء الحساب والصلاحيات في النظام.
   - نقل/إعادة تقديم حقول التشغيل (مركبة/عمولة/هاتف) في إدارة التوصيل.
   - التحقق: تعديل بيانات تشغيل السائق يتم من قناة التوصيل فقط.

4. تحسين التنظيم البنيوي للصفحات الثقيلة:
   - تفكيك أقسام `WarehousePage`, `ProductsPage`, `FinancialPage` إلى Sections واضحة بعناوين ثابتة.
   - التحقق: فصل بصري واضح بدون تغيير الهوية.

5. التجاوب والجداول:
   - إضافة `data-label` في جدول سجل التوصيل داخل `DeliveryBoardPage`.
   - معالجة شريط تبويبات التقارير ليكون مناسبًا للموبايل.
   - التحقق: تجربة واضحة على العرض الضيق.

## التنظيف النهائي (بعد المرحلتين)
- إزالة أي روابط قديمة أو مسارات غير مستخدمة بعد النقل.
- التحقق من عدم وجود تكرار في النصوص/الأعمدة أو تعارض بالروابط.
- مراجعة سريعة للمجلدات المتأثرة لضمان عدم وجود مخلفات.
