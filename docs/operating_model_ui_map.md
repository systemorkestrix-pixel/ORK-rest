# خريطة اللوحات والمسارات والصفحات

هذه الوثيقة تحول النموذج التشغيلي إلى خريطة تنفيذية مرتبطة بالواجهة الحالية، حتى نعرف بدقة ما الذي يجب تعديله مباشرة داخل النظام.

## 1) الهدف
- ربط كل لوحة بمساراتها وصفحاتها الفعلية.
- تحديد مالك كل صفحة داخل النظام.
- منع تكرار الوظائف بين لوحات الإدارة والتنفيذ.
- استخراج قائمة تعديلات واجهة مباشرة قابلة للتنفيذ.

## 2) التطبيقات المعتمدة فعليًا

### أ. لوحة الإدارة
- التطبيق: `Manager Console`
- الجذر: `/console`
- الملكية: الإدارة + التشغيل اليومي

### ب. لوحة المطبخ
- التطبيق: `Kitchen Console`
- الجذر: `/kitchen/console`
- الملكية: تنفيذ المطبخ فقط

### ج. لوحة التوصيل
- التطبيق: `Delivery Console`
- الجذر: `/delivery/console`
- الملكية: تنفيذ السائق فقط

### د. الواجهة العامة
- التطبيق: `Public Interface`
- الجذر: `/order`, `/menu`, `/public/*`
- الملكية: العميل أو الاستخدام العام

## 3) خريطة لوحة الإدارة الحالية

### العمليات
- القناة: `operations`
- المالك التشغيلي: التشغيل اليومي

المسارات الحالية:
- `/console`
  - الصفحة: `ConsolePage` مع لوحة الطلبات النشطة
  - الملكية: تشغيل يومي
- `/console/operations/overview`
  - الصفحة: `ManagerDashboardPage`
  - الملكية: تشغيل يومي
- `/console/operations/orders`
  - الصفحة: `ManagerOrdersPage`
  - الملكية: تشغيل يومي
- `/console/operations/tables`
  - الصفحة: `ManagerTablesPage`
  - الملكية: تشغيل يومي
- `/console/alerts`
  - الصفحة: `ManagerAlertsPage`
  - الملكية: متابعة تشغيلية مركزية

القرار المعتمد:
- الطلبات والطاولات تبقى داخل العمليات.
- التنبيهات لا تبقى كقسم داخل شريط العمليات، بل كمدخل مستقل من الهيدر.
- الصفحة الرئيسية داخل `/console` تبقى لوحة تشغيلية مختصرة وليست بديلًا عن صفحات الأقسام.

### المطعم
- القناة: `restaurant`
- المالك: الإدارة + التشغيل اليومي بحسب الوظيفة

المسارات الحالية:
- `/console/restaurant/menu`
  - الصفحة: `ProductsPage`
  - الملكية: إدارة الأصناف والتسعير

القرار المعتمد:
- المنيو يبقى هنا.
- لا تنقل إدارة المنتجات إلى المطبخ.

### إدارة التوصيل
- القناة: `delivery`
- المالك: الإدارة

المسارات الحالية:
- `/console/delivery/drivers`
  - الصفحة: `DeliveryDriversPage`
  - الملكية: إدارة السائقين والمتابعة
- `/console/delivery/history`
  - الصفحة: `DeliveryHistoryPage`
  - الملكية: مراجعة تشغيلية وإدارية
- `/console/delivery/settings`
  - الصفحة: `DeliverySettingsPage`
  - الملكية: الرسوم والسياسات والتفعيل

القرار المعتمد:
- هذا القسم إداري فقط.
- أي إجراء مثل استلام المهمة أو الانطلاق أو التسليم لا يجب أن يظهر هنا.
- لوحة السائق تبقى منفصلة تمامًا.

### المخزون
- القناة: `warehouse`
- المالك: المخزون + الإدارة

المسارات الحالية:
- `/console/warehouse/stock-ledger`
  - الصفحة: `StockLedgerPage`
  - الملكية: حركة المخزون
- `/console/warehouse/vouchers`
  - الصفحة: `WarehouseVouchersPage`
  - الملكية: السندات
- `/console/warehouse/suppliers`
  - الصفحة: `WarehouseSuppliersPage`
  - الملكية: الموردون

القرار المعتمد:
- كل أعمال المخزون تبقى هنا.
- لا تنقل إلى لوحة العمليات إلا عبر تنبيهات أو ملخصات فقط.

### المالية
- القناة: `finance`
- المالك: المالية + التشغيل اليومي عند الحاجة

المسارات الحالية:
- `/console/finance/dashboard`
  - الصفحة: `FinanceDashboardPage`
- `/console/finance/transactions`
  - الصفحة: `FinanceTransactionsPage`
- `/console/finance/expenses`
  - الصفحة: `FinanceExpensesPage`
- `/console/finance/cash-shift`
  - الصفحة: `CashShiftPage`

القرار المعتمد:
- المالية تبقى مستقلة.
- التسويات المرتبطة بالطلبات يمكن أن تبدأ من التشغيل، لكن شاشة الإدارة المالية نفسها تبقى هنا.

### التحليلات
- القناة: `intelligence`
- المالك: الرقابة والتحليل

المسارات الحالية:
- `/console/intelligence/operational-heart`
  - الصفحة: `OperationalHeartPage`
- `/console/intelligence/reports`
  - الصفحة: `IntelligenceReportsPage`

القرار المعتمد:
- القلب التشغيلي للمراقبة والتحليل السريع.
- التقارير للمراجعة والقراءة وليس للتنفيذ.

### النظام
- القناة: `system`
- المالك: الإدارة التقنية والإدارية

المسارات الحالية:
- `/console/system/users`
  - الصفحة: `SystemUsersPage`
- `/console/system/catalog/products`
  - الصفحة: `ProductsPage`
- `/console/system/roles`
  - الصفحة: `SystemRolesPage`
- `/console/system/settings`
  - الصفحة: `SystemSettingsPage`
- `/console/system/audit-log`
  - الصفحة: `SystemAuditLogPage`

القرار المعتمد:
- النظام يبقى إداريًا.
- لا يجب أن يحتوي أدوات تنفيذ يومي.
- وجود `catalog/products` تحت النظام تكرار وظيفي مع `restaurant/menu` ويحتاج قرار تنظيف.

## 5) خريطة لوحة التوصيل الحالية
- `/delivery/console`
  - الصفحة: `DeliveryPanelPage`

القرار المعتمد:
- هذه اللوحة للتنفيذ فقط.
- لا يجب أن تحتوي إدارة رسوم أو سياسات أو مستخدمين.

## 6) خريطة الواجهة العامة الحالية
- `/order`
  - الصفحة: `PublicOrderPage`
- `/menu`
  - الصفحة: `PublicOrderPage`
- `/public/tables`
  - الصفحة: `PublicTablesPage`

القرار المعتمد:
- هذه الواجهة للعميل أو الطلب العام.
- لا تنقل إليها وظائف الإدارة أو التنفيذ الداخلي.

## 7) التعديلات المطلوبة مباشرة في الواجهة

### أ. تثبيت ملكية الصفحات
يجب اعتماد الملكية التالية في الواجهة:
- الطلبات والطاولات: العمليات
- المنيو: المطعم
- السائقون ورسوم التوصيل والسياسات: إدارة التوصيل
- التنفيذ الفعلي للتوصيل: لوحة السائق فقط
- التحضير والجاهزية: لوحة المطبخ فقط
- التقارير والقلب التشغيلي: التحليلات
- المستخدمون والأدوار والإعدادات والتدقيق: النظام

### ب. إزالة التكرار الوظيفي
العناصر التي تحتاج تنظيفًا أو قرارًا واضحًا:
- `ProductsPage` موجودة تحت:
  - `/console/restaurant/menu`
  - `/console/system/catalog/products`

القرار المقترح:
- اعتماد `restaurant/menu` كواجهة العمل الأساسية
- إبقاء `system/catalog/products` فقط إذا كان لها غرض إداري مختلف وواضح
- وإلا تزال من التنقل اليومي

### ج. تثبيت التنبيهات كمدخل مستقل
- تبقى صفحة التنبيهات في: `/console/alerts`
- لا تعرض كقسم دائم داخل شريط العمليات
- الوصول لها يكون من الهيدر أو من مسارات مباشرة من داخل النظام

### د. فصل المتابعة السريعة عن الصفحات الكاملة
الهيدر يجب أن يحتوي فقط على:
- زر الرئيسية
- زر التنبيهات
- زر حالة النظام أو سجل الاستقرار لاحقًا
- زر حساب المدير لاحقًا
- زر الخروج

ولا يجب أن يتحول الهيدر إلى بديل عن صفحات الأقسام.

## 8) خريطة ما يجب نقله لاحقًا

### حالة الاستقرار
المكان المقترح:
- `/console/system/stability`

الملكية:
- مراقبة تشغيلية مختصرة
- قراءة فقط

### حساب المدير
المكان المقترح:
- `/console/system/account`

الملكية:
- إعدادات شخصية للمدير
- لا تكرر إدارة المستخدمين

## 9) قائمة التنفيذ المباشر للواجهة

### أولوية أولى
- تثبيت أن كل صفحة داخل القناة الصحيحة فقط
- إزالة أي دخول تنفيذي من لوحة الإدارة إلى المطبخ أو السائق
- إزالة التكرار بين أقسام النظام والمطعم عند إدارة المنتجات

### أولوية ثانية
- بناء صفحة `سجل حالة الاستقرار`
- بناء صفحة `حساب المدير`
- ربط الهيدر بهاتين الواجهتين بدل البطاقات العامة الحالية

### أولوية ثالثة
- مراجعة كل زر تنقل داخلي للتأكد من أنه يذهب إلى المالك الصحيح
- مراجعة الرسائل والتسميات لتطابق دور المستخدم النهائي

## 10) القرار التنفيذي
هذه هي الخريطة المعتمدة لبناء الواجهة من الآن:
- Manager Console = إدارة + تشغيل يومي
- Kitchen Console = تنفيذ مطبخ فقط
- Delivery Console = تنفيذ توصيل فقط
- Public Interface = عميل فقط

وكل تعديل جديد في الواجهة يجب أن يمر عبر هذه القاعدة قبل تنفيذه.
