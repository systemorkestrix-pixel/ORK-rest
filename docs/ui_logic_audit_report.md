# تقرير تدقيق منطق التوزيع الوظيفي (UI Logic Audit Report)

تاريخ التقرير: 2026-03-15

## 1) الهدف
تحليل توزيع الأدوات والجداول والإجراءات عبر الواجهات وفق منطق العمل التشغيلي والإداري، ورصد العناصر الدخيلة أو غير المنطقية واقتراح نقلها إلى موقعها الصحيح.

## 2) منهجية الفحص
- مراجعة صفحات الإدارة والتشغيل والواجهات العامة.
- تتبع الأدوات التي تغيّر إعدادات تشغيلية أو سياسات.
- تتبع الجداول التي تخص نطاقًا إداريًا لكنها تظهر في واجهات تشغيلية.
- تصنيف العناصر وفق: إدارة (Management) مقابل تشغيل (Operations).

## 3) عناصر تحتاج نقل (مقترحات واضحة)

### 3.1 رسوم وسياسات التوصيل
- **الموقع الحالي:** `src/modules/system/settings/SettingsPage.tsx`
  - قسم "إعدادات التوصيل العامة" (رسوم التوصيل، الحد الأدنى، التبليغ التلقائي).
- **الموقع المقترح:** قناة التوصيل الإدارية `/console/delivery/*`.
- **المبرر:** هذه سياسات تشغيلية تخص التوصيل مباشرة؛ وجودها ضمن إعدادات النظام يشتت المستخدم ويخالف مبدأ ملكية القسم.

### 3.2 رابط تعديل الرسوم داخل إدارة التوصيل
- **الموقع الحالي:** `src/modules/delivery/drivers/DeliveryTeamPage.tsx` يحتوي رابطًا إلى `/console/system/settings`.
- **الموقع المقترح:** بعد نقل إعدادات التوصيل، يجب أن يشير الرابط إلى صفحة إعدادات التوصيل داخل قناة التوصيل.
- **المبرر:** الحفاظ على مسار منطقي داخل نفس القناة.

### 3.3 بيانات تشغيل السائق (مركبة/عمولة/هاتف)
- **الموقع الحالي:** `src/modules/system/users/UsersPage.tsx`.
- **الموقع المقترح:** فصل طبقة البيانات التشغيلية للسائق إلى صفحة إدارة التوصيل مع بقاء إنشاء الحساب/الدور ضمن النظام.
- **المبرر:** تفاصيل تشغيل السائق تخص إدارة التوصيل، بينما النظام يجب أن يحتفظ بهيكل الحساب والصلاحيات فقط.

## 4) عناصر تحتاج مراجعة ملكية (قد تكون صحيحة لكن تتطلب قرار)

### 4.1 تسوية التوصيل داخل أوامر العمليات
- **الموقع الحالي:** `src/modules/operations/orders/OrdersPage.tsx` يحتوي زر "تسوية التوصيل".
- **الاقتراح:** إبقاء عرض حالة التسوية داخل العمليات، ونقل إجراء التسوية الفعلي إلى Finance → تسويات التوصيل.
- **المبرر:** التسوية فعل مالي، لكن عرض الحالة في العمليات ضروري لإدارة الطلب.

### 4.2 لوحة إدارة السائقين تعرض ملخصات تشغيلية
- **الموقع الحالي:** `src/modules/delivery/drivers/DeliveryTeamPage.tsx`.
- **الاقتراح:** هذا مناسب إدارياً، لكن يُفضَّل تجميع الملخصات التشغيلية في بطاقة مستقلة لتقليل التشويش.

## 5) عناصر منطقها صحيح (لا نقل مطلوب)
- تقارير وإيرادات التوصيل ضمن Finance (`src/modules/finance/transactions/FinancialPage.tsx`) — صحيحة لأنها مالية.
- عرض رسوم التوصيل داخل Delivery Console للعرض فقط — صحيح لأنها معلومة تشغيلية.
- إعدادات الحساب والجلسات والنسخ الاحتياطي ضمن النظام — صحيحة لأنها نظامية.

## 6) التوصيات التنفيذية
1. نقل إعدادات التوصيل (رسوم/حد أدنى/تبليغ تلقائي) إلى صفحة ضمن قناة التوصيل الإدارية.
2. تعديل روابط إدارة التوصيل لتبقى داخل القناة بعد النقل.
3. فصل بيانات تشغيل السائق عن إدارة المستخدمين العامة (إنشاء حساب يبقى في النظام، تشغيل السائق ينتقل للتوصيل).
4. حسم ملكية إجراء تسوية التوصيل (توصية: تنفيذ مالي داخل Finance مع إبقاء عرض الحالة في Operations).
